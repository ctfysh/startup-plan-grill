# startup-plan-grill 设计文档 (v3)

## 1. 概述

startup-plan-grill 是一个交互式创业教练技能。它用苏格拉底式的发问，引导一个零基础的菜鸟从模糊想法起步，逐步打磨出完整的创业计划。最终输出两份文档：过程记录 + 10章创业计划书。

核心原则：不讲课，只发问。每个问题来自 7 本创业经典的核心框架，迫使用户自己思考出答案。

## 2. 覆盖书籍与核心内容

|------|----------|---------------|

## 3. v2 → v3 改进说明

### v2 主要问题

| 问题 | 表现 | v3 修正 |
|------|------|---------|
| 顺序错位 | BMG 画布(Stage 2)在 TBM 机会测试(Stage 3)之前 | BMG 移到 Stage 2（先测试机会，再设计模式） |
| 内容重叠 | LS 创新核算(Stage 4)与 LS2 创新核算(Stage 5)重复 | LS 的创新核算留在 Stage 3；LS2 的 3 层次创新核算融入 Stage 4 组织治理 |
| 内容割裂 | Z2O 销售(Stage 1)与 LS 增长引擎(Stage 4)都在讲增长 | 合并为 Stage 4 增长部分 |
| 团队位置 | Z2O 团队(Stage 1)放在太早，缺乏组织设计上下文 | 移到 Stage 4，与 LS2 组织设计放在一起 |
| 缺少财务/融资 | 无财务规划内容 | 新增 Stage 5，用两本新书覆盖 |

### 跨书张力处理

- Z2O vs LS 哲学冲突: Z2O 主张"秘密→垄断"的确定论规划路线；LS 主张"假设→验证"的试错路线。处理方式：Z2O 作为战略假设（Stage 0），LS 作为验证方法（Stage 3），二者不矛盾。先提出垄断假设，再用精益方法验证。
- TBM vs BMG 侧重不同: TBM 侧重外部机会评估，BMG 侧重内部模式设计。处理方式：先评估机会（Stage 1），确认值得做之后，再设计模式（Stage 2）。
- LS vs LS2 重复: LS 的创新核算(3步)和 LS2 的创新核算(3层次)是同一概念的不同细化。处理方式：LS 的 3 步核算放在 Stage 3（MVP 验证阶段）；LS2 的 3 层次放在 Stage 4（组织治理阶段）。

## 4. 创业计划书 6 阶段总流程图

```
用户提出想法
    ↓
Stage 0: 战略定位 / 秘密·反主流·垄断·幂次法则·特斯拉7问 [Z2O]
    ↓ [检查点] 值得深入吗？
Stage 1: 机会评估 / 7领域全面体检 [TBM]
    ↓ [检查点] 机会成立吗？
Stage 2: 商业模式设计 / 画布9模块·ERRC·SWOT [BMG]
    ↓ [检查点] 模式完整吗？
Stage 3: 精益验证 / 假设→MVP→BML→转型判断 [LS]
    ↓ [检查点] 心态准备好了吗？
Stage 4: 引擎与组织 / 增长·销售·团队·治理·5原则 [Z2O + LS + LS2]
    ↓ [检查点] 组织能运转吗？
Stage 5: 财务与融资 / 报表·比率·架构·融资·估值 [财报 + Venture Deals]
    ↓
整合输出 → 创业计划书 .md (10 章)
```

**为什么是这个顺序？**

1. 先想清楚方向再测试（Stage 0 → Stage 1）：先回答"这到底是个什么想法"，再深入测试。Z2O 的"秘密"提供思考框架，TBM 的 7 领域提供测试工具。
2. 先确认机会再设计模式（Stage 1 → Stage 2）：如果机会不成立，模式再漂亮也没用。
3. 先设计模式再实验验证（Stage 2 → Stage 3）：有了完整的模式假设，才知道哪些假设最需要验证。
4. 先验证了再考虑规模化（Stage 3 → Stage 4）：未经验证的模式不值得增长和建组织。
5. 一切验证完后才做财务预测和融资（Stage 4 → Stage 5）：没有经过验证的财务预测是纯粹的幻想。

## 5. 六阶段详细设计

### Stage 0: 战略定位 / 这到底是个什么想法？

**目标**: 帮用户用 Z2O 的框架审视自己的想法，判断是否值得深入。

**时长预估**: 5-8 轮问答  
**参考**: 从0到1 Ch1 + Ch3 + Ch4 + Ch5 + Ch6 + Ch7 + Ch13

#### 0A / 热身：一句话想法

| # | 问题 | 分支逻辑 |
|---|------|---------|
| 0.1 | 用一句话描述你的创业想法：你为谁解决了什么问题？ | 说不清 → 追问"客户是谁"和"什么问题"；说清了 → 进 0.1a |
| 0.1a | 你这个想法是"从0到1"（创造一种前所未有的东西）还是"从1到n"（复制已有的成功模式，但做得更好或更便宜）？ | 从1到n → 追问"你凭什么认为你能比现有玩家做得更好？如果只是更好一点，用户为什么要换？"；从0到1 → "那意味着你相信一个别人没看到的秘密，进0.2" |

#### 0B / 秘密与反主流观点

| # | 问题 | 分支逻辑 |
|---|------|---------|
| 0.2 | 在什么重要问题上，你与绝大多数人有不同看法？ | 回答与创业无关 → 追问"这对你的生意意味着什么"；说不出 → 红灯"没有独特观点的生意很难垄断" |
| 0.3 | 你这个想法是基于一个什么样的"秘密"？是别人不相信但你知道的技术真相，还是别人没注意到的用户需求空白？ | 自然秘密→走技术路线；人的秘密→走需求路线；没有秘密→建议重新审视 |
| 0.3a | 你怎么看待竞争？很多人说"有竞争说明市场好"，但也有人说"竞争是利润的毁灭者" | 正面看待竞争 → "小心：如果赛道已经挤满了人，你的'秘密'可能根本不是秘密"；负面看待竞争 → "你打算怎么避开竞争，创造垄断？进0.4" |

#### 0C / 垄断与护城河

| # | 问题 | 分支逻辑 |
|---|------|---------|
| 0.4 | 你的产品/服务靠什么保护自己不被模仿？选出最匹配的一项：专利技术 / 网络效应 / 规模经济 / 品牌优势，或者以上都不是？ | 以上都不是 → 严重警告"没有护城河的生意容易被巨头碾压" |
| 0.5 | 你的起步市场是多大？具体说，你第一个打入的市场是多小的？ | 市场太大 → 警告"大市场竞争惨烈，建议从细分小市场开始"；太小且无增长可能 → 警告"天花板太低" |
| 0.6 | 你的终极市场会比现在大 10 倍吗？ | 不能 → 提醒"投资机构会关心增长空间" |
| 0.6a | 你是在创造一个全新的品类（像特斯拉开创了电动豪华车市场），还是在一个成熟的品类里和现有玩家抢份额？ | 创造新品类 → 需要教育市场，资源够吗？；成熟品类竞争 → "你觉得现在的领头羊最怕什么？你有办法让它睡不着吗？" |

#### 0D / 幂次法则与聚焦

| # | 问题 | 分支逻辑 |
|---|------|---------|
| 0.7 | 如果有且只能做一件事来让这个创业成功，那件事是什么？ | 说不清 → "幂次法则：最重要的事胜过其余总和"；说多个 → 追问哪个最决定成败 |
| 0.8 | 你对未来的态度是什么？是认为未来可以被预测和规划，还是走一步看一步？ | 明确乐观 → 适合精细规划；不明确 → 需要先做更多调研 |

#### 0E / 特斯拉7问全景扫描（Z2O Ch13）

在深入各阶段之前，用彼得·蒂尔的"特斯拉7问"快速扫描你的想法的完整性。这7个问题覆盖了后面所有Stage要深挖的方向：

| # | 问题 | 对应 & 状态 |
|---|------|-------------|
| 0.9 | 技术：你的产品在核心技术上有突破性进步吗？进步幅度是10倍级的还是10%级的？ | 对应 Stage 1/2 (待定) |
| 0.10 | 时机：为什么现在是做这件事的最佳时机？以前为什么没人做？ | 对应 Stage 1 (待定) |
| 0.11 | 垄断：你起步时能在一个小市场占据大份额吗？（参考 0.4-0.6） | 已在 0C 覆盖 (已覆盖) |
| 0.12 | 团队：你和你团队有做这件事的独特优势吗？ | 对应 Stage 4 (待定) |
| 0.13 | 销售：你怎么让用户知道并买到你的产品？ | 对应 Stage 4 (待定) |
| 0.14 | 持久：10年后你的市场地位还稳吗？什么力量可能颠覆你？ | 对应 Stage 5（财务中的风险评估）(待定) |
| 0.15 | 秘密：你发现了什么别人没看到的独特机会？（参考 0.3） | 已在 0B 覆盖 (已覆盖) |

用户对每个"待定"问题做一句话回答。说不清的标记为"红灯"，在后面对应的Stage重点追问。全部答完后进入下一Stage。

**Stage 0 输出**: 一句话价值主张 + 秘密声明 + 护城河策略 + 聚焦方向 + 特斯拉7问状态灯

**检查点**: 在进入机会评估前，问用户"基于以上7问，你觉得这个想法值得花时间深入测试吗？还是需要先做更多调研/放弃？" 用户选择继续则进入 Stage 1，否则建议先补充调研。

### Stage 1: 机会评估 / 这个想法经得起检验吗？

**目标**: 用 TBM 的 7 领域模型做全面体检，判断机会是否值得追求。

**时长预估**: 10-15 轮问答  
**参考**: 如何测试商业模式 Ch1-Ch8

#### 1A / 市场宏观

| # | 问题 | 来源 |
|---|------|------|
| 1.1 | 你进入的市场总规模有多大？（人数/金额/数量）如果不知道，打算怎么估算？ | TBM Ch1 |
| 1.2 | 这个市场过去 3 年增长多快？未来 3 年预测增速？ | TBM Ch1 |
| 1.3 | 有哪些大趋势在推动或阻碍这个市场？（人口/技术/社会/政策/环境） | TBM Ch1 |

#### 1B / 市场微观 + 移情图

| # | 问题 | 来源 |
|---|------|------|
| 1.4 | 谁是你主要的目标客户？描述一个典型用户的画像（年龄、身份、场景、痛点） | TBM Ch2 + 移情图 |
| 1.5 | 这个客户现在用什么替代方案解决他的问题？他对现状满意吗？ | TBM Ch2 |
| 1.6 | 客户愿意为你的解决方案付多少钱？这个价格覆盖得了成本吗？ | TBM Ch2 |
| 1.7 | （移情图追问）客户嘴上说想要什么，和他实际做的有什么不一致？不一致的地方往往就是机会 | BMG 移情图 |

#### 1C / 行业宏观（五力分析）

| # | 问题 | 来源 |
|---|------|------|
| 1.8 | 这个行业新进入者容易进来吗？进入壁垒有哪些？ | TBM Ch4 |
| 1.9 | 供应商的议价能力强吗？你依赖少数的供应商还是有很多选择？ | TBM Ch4 |
| 1.10 | 购买者（你的客户）议价能力如何？他们能轻松换别家吗？ | TBM Ch4 |
| 1.11 | 替代品的威胁有多大？客户有别的办法解决同样的问题吗？ | TBM Ch4 |
| 1.12 | 行业目前的竞争有多激烈？有几家主要玩家？ | TBM Ch4 |

#### 1D / 行业微观（竞争优势）

| # | 问题 | 来源 |
|---|------|------|
| 1.13 | 你的利基（niche）在哪？能在行业中找到一个可防御的细分位置吗？ | TBM Ch5 |
| 1.14 | 你的竞争优势能持续多久？靠什么保持？（专利/品牌/规模/数据/网络效应） | TBM Ch5 |
| 1.15 | 你算过基本单位经济学吗？单个客户卖多少，直接成本（COGS）是多少？毛利率够不够覆盖后面的销售和管理费用？ | TBM Ch5（注意：CAC/CLV 将在 Stage 4 中深入计算） |

#### 1E / 使命与个人

| # | 问题 | 来源 |
|---|------|------|
| 1.16 | 你为什么做这件事？纯粹赚钱还是更深层的使命驱动？ | TBM Ch6 |
| 1.17 | 如果 5 年不赚钱，你还会继续做吗？ | TBM Ch6 |
| 1.18 | 你愿意为这个创业付出多大代价？（时间、金钱、生活方式） | TBM Ch6 |

#### 1F / 执行能力

| # | 问题 | 来源 |
|---|------|------|
| 1.19 | 这个行业成功的关键要素是什么？（技术/渠道/品牌/资金/人才） | TBM Ch7 |
| 1.20 | 你和团队在哪些关键要素上有优势？哪些是短板？ | TBM Ch7 |
| 1.21 | 短板能通过招人/学习/合作补齐吗？ | TBM Ch7 |

#### 1G / 关系网络

| # | 问题 | 来源 |
|---|------|------|
| 1.22 | 你和团队在这个行业里有人脉吗？能打开关键渠道吗？ | TBM Ch8 |
| 1.23 | 有没有潜在的导师/顾问可以帮你少踩坑？ | TBM Ch8 |

**Stage 1 输出**: 7 领域综合评分 + 机会风险清单 + "做 or 不做"决策辅助

**检查点**: 问用户"综合7领域的评估结果，这个机会值得你投入时间/金钱/精力吗？" 如果用户犹豫，需要先解决具体短板再推进。

### Stage 2: 商业模式设计 / 怎么做成这件事？

**目标**: 设计完整的商业模式画布，匹配模式类型，评估短板。

**时长预估**: 12-18 轮问答  
**参考**: 商业模式新生代 全书（画布9模块 + 5模式类型 + SWOT + 蓝海战略ERRC）

#### 2A / 画布 9 模块（依次扫描）

按顺序逐格完成，每格确认后再进下一个：

| # | 模块 | 关键问题 |
|---|------|---------|
| 2.1 | CS 客户细分 | 我们在为谁创造价值？可分为几个细分群体？谁最重要？ |
| 2.2 | VP 价值主张 | 我们帮客户解决什么问题？满足什么需求？这是"救命的药"还是"维生素"？ |
| 2.3 | CH 渠道通路 | 客户希望怎么接触到我们？从认知到评估到购买到传递到售后，哪个阶段最薄弱？ |
| 2.4 | CR 客户关系 | 每个细分期待什么类型的关系？获取成本高还是留存成本高？ |
| 2.5 | R$ 收入来源 | 客户愿意为什么价值付钱？怎么付（买断/订阅/租赁/中介费/广告）？定价策略？ |
| 2.6 | KR 核心资源 | 最需要什么才能让生意运转（实体/知识/人力/资金）？ |
| 2.7 | KA 关键业务 | 每天必须做的核心事是什么（生产/问题解决/平台网络）？ |
| 2.8 | KP 重要合作 | 关键合作伙伴是谁？他们提供什么资源？ |
| 2.9 | C$ 成本结构 | 最贵的成本是什么？成本驱动还是价值驱动？ |

#### 2B / 模式类型匹配

画布填完后，追问：

| # | 问题 |
|---|------|
| 2.10 | 你的模式最接近哪种类型：分拆型（不同业务分离）/ 长尾型（大量小众产品）/ 多边平台型（连接两个群体）/ 免费型（交叉补贴）/ 开放型（外部协作）？ |
| 2.11 | 你是做客户原来自己做的事（替他们省事），还是做他们做不到的事？ |

#### 2C / SWOT 评估

| # | 问题 |
|---|------|
| 2.12 | 你的画布每个模块，各自的优势/劣势/机会/威胁是什么？ |
| 2.13 | 有没有哪个模块之间明显不匹配？（如 VP 和 R$ 对不上，或 KA 和 C$ 矛盾？） |
| 2.14 | 如果只能改一个模块来最大程度降低风险，改哪个？怎么改？ |

#### 2D / 蓝海战略（ERRC 四项行动框架）

基于你对行业竞争格局的理解（Stage 1 输出），用蓝海战略的 ERRC 框架重新审视你的价值主张：

| # | 问题 | 说明 |
|---|------|------|
| 2.15 | 消除（Eliminate）：行业里哪些被当作理所当然的元素，其实应该被消除？（因为它们不再创造价值） | 如：传统酒店的礼宾台。Airbnb 消除了它。 |
| 2.16 | 降低（Reduce）：哪些元素应该显著降低标准，因为过度设计？（用户并不在乎） | 如：廉价航空的机上服务 |
| 2.17 | 提升（Raise）：哪些元素应该大幅提升到行业标准之上？（用户极度在乎） | 如：西南航空的准点率 |
| 2.18 | 创造（Create）：行业从未提供过但用户会喜欢的全新元素是什么？ | 如：iPhone 创造触摸屏交互 |
| 2.19 | 综合以上四项：你的"价值曲线"和竞争对手比有什么本质不同？ | 如果和对手走同一条曲线，你只是在复制 |

**Stage 2 输出**: 完整商业模式画布（文本化）+ 模式类型 + SWOT + ERRC 价值曲线 + 短板清单

## 7. 实施方式

**互动模式**: 纯代理交互式。不涉及 CLI 脚本、不需要安装依赖。

代理加载此 skill 后，按以下流程工作：

1. 让用户用一句话描述创业想法
2. 按 Stage 0 到 1 到 2 到 3 到 4 到 5 依次进行
3. 每个 Stage 内部：每次只问一个问题，根据回答选分支
4. Stage 结束给出该阶段总结（结构化的文本输出）
5. 进入下一 Stage 前问用户"准备好继续了吗？"
6. 用户可随时要求"回看"上一个阶段的总结
7. 全部完成后输出两份文档：过程记录 + 10章创业计划书

**用户可选操作**:
- "重新回答 X 阶段" → 回到那个阶段重头再来
- "跳过 Y 阶段" → 跳过不适用的部分（特别是 Stage 5E 估值条款部分）
- "给我看目前的计划书草稿" → 随时汇总已有内容

## 7. 用户故事示例

技能包 `examples/` 目录下包含 6 个示例对话文件（Stage-0.md 至 Stage-5.md），使用"校园二手教材 C2C 交易平台"作为统一示例场景，展示了每道题的 Story/Demo/Acceptance/Error/Edge 五段式回答。总篇幅约 2073 行，覆盖全部 136 道问题。

用途：
- **开发者参考**：展示 agent 在每道题上的预期行为，尤其是 Demo 段落的追问方式和 Error 段落的错误处理
- **验收标准**：Acceptance 段落可直接转化为集成测试用例
- **边缘情况参考**：Edge 段落覆盖了极端场景的处理策略

注意事项：示例对话只是示例，实际对话中用户回答可能完全不同。agent 不应背故事中的答案，而应基于 REFERENCE.md 的 4 级追问规则动态应对。

## 8. 与 grill-me 的区别

| 维度 | grill-me | startup-plan-grill |
|------|----------|--------------------|
| 目标 | 打磨现有计划/设计 | 从零构建创业计划书 |
| 知识来源 | 用户自己的计划 | 7 本创业书的框架体系 |
| 输出 | 共享理解 | 过程记录 + 结构化创业计划书 .md (10 章, 7000-11000字) |
| 阶段 | 无固定结构 | 6 Stage 递进 |
| 问题来源 | 代理自己推理 | 书中已验证的框架问题 |
| 广度/深度 | 纵深追问不限方向 | 广度优先，覆盖所有主要维度 |

## 9. 边界与排除

- 不做：编写具体财务模型（Excel/Google Sheets）、制作 pitch deck 视觉设计、法律文件撰写
- 不做：替代专业商业计划书顾问、会计师或律师
- 不做：对用户想法做"好/坏"判断，只帮用户自己想清楚
- 可以：根据用户回答指出逻辑矛盾和盲区
- 可以：在用户多次矛盾回答后提出红色警报
- 可以：对 Stage 5E 部分提示"这部分专业性较强，可咨询专业律师/会计师"
