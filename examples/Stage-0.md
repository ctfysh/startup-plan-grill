# Stage 0: User Stories - 战略定位

> 本文件包含 Stage 0（战略定位）全部 15 个问题 + 6 个子问题的用户故事。
> 每个问题包含 Story / Demo / Acceptance / Error / Edge 五个段落。
> 格式统一，开发人员可直接按此实现 agent 行为。
>
> 跨书张力提示：Stage 0 全部来自《从0到1》(Z2O)，其底层哲学是"秘密→垄断"的确定论规划路线。
> 你找到一个别人没发现的秘密，然后设计一个垄断生意把它变现。这与《精益创业》(LS) 的
> "假设→实验→验证"试错路线存在根本张力。处理原则：Z2O 提供战略假设（Stage 0 产出的
> 是"待验证的假设"），LS 提供验证方法（Stage 3 检验这些假设）。两者不矛盾，但 agent
> 在 Stage 0 追问时应注意不要过度承诺"确定性"，对过于自信的用户可提醒"这目前只是假设，
> 我们将在 Stage 3 设计实验来验证它"。

---

## 0A / 热身：一句话想法

### Q0.1 用一句话描述你的创业想法：你为谁解决了什么问题？

**Story**: 用户坐在屏幕前想了半分钟，打出一段："我做了一个校园二手教材 C2C 平台，专门帮大学生买卖二手教材。帮他们解决每学期教材费太贵、用一学期就闲置的问题。"说完自己又补了一句："差不多就这个意思。"

**Demo**: 代理先确认"客户和问题都明确了"，然后追问："你说的'大学生'具体是指什么群体？大一新生还不懂要买二手只能挨宰，还是大二大三的老生对价格更敏感？"用户回答后，再追问："好，记下了。'教材费太贵'这个问题你现在有数据支撑吗，还是你观察到的一个现象？"

**Acceptance**: 回答同时包含两个要素：明确的客户群体（谁）和具体的痛点（什么问题）。客户不是"所有人"，问题不是"效率低"这种空泛描述。代理能据此画出一幅清晰的需求画面。

**Error**: "我想做一个校园二手交易平台"。只有what没有who和why。代理追问："这个平台为谁服务？解决他们的什么具体问题？如果说不出具体的客户和场景，这个想法目前还只是一个方向不是一个方案。"

**Edge**: 用户说了一长段话（1分钟以上）包含了多个客户群体和多个问题。代理不打断，听完后说："我听到了好几个方向：帮学生买教材、帮毕业生卖旧书、帮考研党找专业课资料。核心是哪一个？用一句话把最核心的那个说清楚。"如果用户坚持认为"这些都是一个东西的不同面"，代理可追问："如果今天只能选一个客户群体先服务，你选谁？"

> Z2O vs LS 张力注：Q0.1 是整场对话的"入口题"。Z2O 框架要求用户一开始就能清晰定义"秘密"的服务对象。代理在此处应保持中立：不评判想法好坏，只确认用户自己是否想清楚了。这个初步定义将在 Stage 3 被转化为可测试的假设。

### Q0.1a 你这个想法是"从0到1"（创造一种前所未有的东西）还是"从1到n"（复制已有的成功模式，但做得更好或更便宜）？

**Story**: 用户想了想说："应该是从1到n吧。市面上已经有类似的二手教材交易渠道了。闲鱼、学校书店、学长学姐私聊群。但它们要么太杂、要么收购价太低、要么效率太低。我做一款专门针对本校的、更可信的版本，大学生在微信小程序上就能完成交易。"

**Demo**: 代理根据回答走分支。如果用户说"从1到n"，追问："你凭什么认为你能比闲鱼做得更好？闲鱼有海量用户和流量优势。你需要在信任和便利之外再给一个他们必须用你的理由。"如果用户说"从0到1"，追问："那意味着你相信了一个别人没看到的秘密。是什么秘密？我们进下一题细聊。"

**Acceptance**: 用户诚实地评估了自己的想法，没有硬说自己是在"创造新品类"以显得更酷。对于"从1到n"的情况，用户提出了除价格之外的差异化理由（比如体验、落地服务、特定场景优化）。对于"从0到1"的情况，用户能初步说出那是什么"秘密"。

**Error**: "我觉得两者都有吧"。用户试图折中。代理不让混过去："选一个最接近的。任何创业同时都有创新和复制的成分，但核心倾向是哪一个？如果核心是解决一个别人没注意到的问题，就是0到1；如果核心是比现有方案做得更好更便宜，是1到n。"

**Edge**: 用户对"0到1"和"1到n"的概念本身困惑，反问"什么叫前所未有？世界上没有绝对的新东西"。代理用例子解释："特斯拉在电动车上是从0到1，之前没有电动豪华车品类。小米做手机是从1到n，当时已经有iPhone和三星了，但小米用性价比打出了自己的位置。"解释后让用户重新选择。

## 0B / 秘密与反主流观点

### Q0.2 在什么重要问题上，你与绝大多数人有不同看法？

**Story**: 用户说："现在大家都在说电子教材会取代纸质教材，大学迟早不需要纸质书了，但我认为恰恰相反。电子教材在大学生群体里使用率其实很低。学生还是习惯在纸上划重点、做笔记。纸质教材不仅不会消失，二手教材交易反而会随着教材越来越贵变得更重要。我觉得数字化没有消灭纸质书，而是让二手纸质教材的需求更集中了。"

**Demo**: 代理先判断这个回答是否与创业相关，本例明显相关。代理追问："这个观点对你的生意意味着什么？如果你的核心假设是'纸质教材不会消失'，那二手教材交易就是一个持续的需求。但如果电子教材某天真的普及了，你的生意可能就消失了。你想过这个风险怎么对冲吗？"

**Acceptance**: 用户给出了一个真正的"反主流"观点。不是人云亦云，不是媒体里的流行共识，而是自己独立思考的结果。这个观点和用户的创业项目之间有逻辑关联。它不需要"正确"，但需要"不同"。

**Error**: "我认为教育应该改革"。回答与创业项目毫无关系。代理不否定这个观点本身，但追问："这个教育观点和你做校园二手教材平台之间有什么关系？如果你的创业是帮大学生买卖教材，你需要在这个领域有与众不同的看法。你在这个行业里观察到什么别人没注意的事？"

**Edge**: 用户想了很久说"我好像没什么和别人不同的看法"。代理给出警告（"没有独特观点的生意很难垄断"），但不否定，而是引导："不一定非要石破天惊。想想你决定做这件事的起因。当时你看到了什么别人忽略的信号？或者你在行业里发现了一个普遍存在的但没人指出的矛盾？"

### Q0.3 你这个想法是基于一个什么样的"秘密"？是别人不相信但你知道的技术真相，还是别人没注意到的用户需求空白？

**Story**: 用户说："我觉得是个需求空白。现在市面上所有二手交易平台都是给大众准备的。闲鱼不做校园细分、学校书店收购价极低。大学生其实只需要一个简单的功能：找到自己这学期要用的、同校学长学姐卖的教材。但这个需求被所有人忽略了，因为大众市场更大、利润更高。"

**Demo**: 代理确认这是"人的秘密"（需求发现），然后推动用户说出更具体的发现："你说的这个需求被忽略，有证据吗？比如你去访谈过多少大学生，他们明确表达了买教材太贵的不满？还是你基于自己的经验推断的？"

**Acceptance**: 用户能清晰区分自己的"秘密"属于哪一类。技术秘密（我知道一个别人不知道的技术可能性）还是人的秘密（我注意到一个别人没看到的需求）。更进一步的优质回答会附带"我凭什么说这是秘密"的证据或经历。

**Error**: "我没有秘密，就是觉得这个方向有机会"。用户回答里没有"秘密"的成分。代理说："彼得·蒂尔说每个成功的创业都建立在一个秘密之上。一个别人不知道或不相信的事实。如果连你自己都不知道这个秘密是什么，你的生意会非常脆弱。建议你重新审视：你的想法中有没有什么是'你以为别人不知道'的东西？"

**Edge**: 用户把"秘密"和"反主流观点"搞混了，说"我反主流观点那个就是我的秘密"。代理区分两者："反主流观点是你和别人的世界观差异，秘密是具体的一个尚未被充分利用的认知。比如'我觉得纸质教材不会消失'是观点，'我发现武大一个年级2000人每学期教材费平均300元但毕业后全部闲置'是秘密。这件事真实存在但大公司没注意到。"

> Z2O vs LS 张力注："秘密"是 Z2O 的核心理念。它假设确实存在一个客观的、可被发现的秘密，找到它就能建立垄断。LS 对"秘密"持怀疑态度：你所谓的"秘密"只是一个未经检验的假设。Stage 0 agent 在此处不必引入 LS 视角，但记录用户的"秘密声明"，留到 Stage 3 转换成可测试假设。

### Q0.3a 你怎么看待竞争？很多人说"有竞争说明市场好"，但也有人说"竞争是利润的毁灭者"

**Story**: 用户说："我以前也觉得有竞争说明这个赛道没错，但现在看法变了。我看到那些扎堆竞争的行业，外卖、打车，最后大家都赚不到钱。我更想找到一个没人抢的角落，做一个小而美的生意。"

**Demo**: 用户给出了"负面看待竞争"的回答。代理追问："你说得对。你打算怎么具体避开竞争？是做一个现有大厂看不上的小市场，还是做一个他们做不了的重服务生意？给我一个你'避开竞争'的具体策略。"

**Acceptance**: 用户不是条件反射地说"有竞争是好事"也不是一味恐惧竞争。用户能说出自己对竞争的具体态度，以及这个态度如何影响自己的创业策略。负面看待竞争的，能说出具体的避让方式；正面看待竞争的，也能说出自己在竞争中脱颖而出的独特优势。

**Error**: "有竞争说明市场好啊，大家都在做说明需求真实"。这是彼得·蒂尔明确批评的"从众心理"。代理给出警告："小心。如果赛道已经挤满了人，你的'秘密'可能根本不是秘密。彼得·蒂尔说'竞争是给失败者的'。竞争激烈的行业利润率趋近于零。你确定你的优势足以在红海中杀出来吗？"

**Edge**: 用户说"我没有竞争，我是独家的"但是说不出为什么别人不进来。代理追问："如果你现在做这个赚钱了，什么阻止一个有钱有团队的大公司在6个月内复制你？如果答不出来，你所谓的'没有竞争'只是暂时的。"

## 0C / 垄断与护城河

### Q0.4 你的产品/服务靠什么保护自己不被模仿？选出最匹配的一项：专利技术 / 网络效应 / 规模经济 / 品牌优势，或者以上都不是？

**Story**: 用户说："网络效应。我们平台上一个学校的卖家越多、教材种类越丰富，买家就越愿意来这里搜书。买家越多，卖家也越愿意来这里挂书。两边互相吸引，后来者很难打破这个循环。而且每个学校是一个独立的网络节点，先占住的学校后来者很难抢走。"

**Demo**: 代理先肯定选择，然后引导用户深入思考这个护城河的启动问题："网络效应是个很好的护城河，但它启动最困难。先有鸡还是先有蛋？你打算怎么跨过冷启动阶段？在只有几十本书挂上去的时候，网络效应还没起效，你靠什么留住第一批用户？"

**Acceptance**: 用户选择了一种护城河类型并给出了具体解释。不是随便选一个词，而是真实理解了这个壁垒如何在自己的生意中起作用。

**Error**: "以上都不是"。代理发出严重警告："没有护城河的生意就像在露天市场摆摊，今天生意好明天就会有人在你隔壁开一家一模一样的。巨头要碾压你只需要动动手指。你至少需要一种壁垒，技术上、规模上、品牌上、还是用户网络上的。不然这个生意经不起推敲。"

**Edge**: 用户选了多种（"我既有专利又有品牌又有网络效应"）。代理不否定，但追问："这些不能都作为主要壁垒。如果现在只能选一个最核心的，它是哪一个？因为不同的壁垒需要不同的资源投入和策略，得先知道哪个是你的命门。"

### Q0.5 你的起步市场是多大？具体说，你第一个打入的市场是多小的？

**Story**: 用户说："我打算从武汉大学开始。大概有5万学生，每年春秋两季是换书高峰。开学前两周所有人都需要买教材。而且武大学生集中在文理学部、工学部、信息学部三个区域，配送成本很低。"

**Demo**: 代理表示认可，然后追问经营层面的问题："好，非常具体的起步市场。武大5万学生，假设每学期人均教材支出300元，全校二手教材流转的潜在市场规模大概1500万一年。你估计第一学期能拿下百分之几的交易量？算算起步阶段的流水预期是多少。"

**Acceptance**: 用户给出了具体的、有边界的起步市场。不是一个城市或一个省，而是可触达的、有明确边界的一个细分入口。用户能说出"为什么从这个市场切入"而非从别的市场。

**Error**: "全国大学生教材市场有上百亿"。代理发出警告："大市场竞争惨烈。彼得·蒂尔说每个垄断生意都从一个小市场起步。eBay从收藏品小圈子开始、Facebook从哈佛一个校园开始。如果你一开始就想做大市场，你哪来的资源和大厂正面竞争？请重新聚焦：你第一个能'占据'的市场是什么，不是一个'参与'的市场。"

**Edge**: 用户说了非常具体的市场但确实太小（如"我只做我们系一个年级的教材交易"）。代理判断天花板："一个年级就算1000人，每学期人均交易50元，一年流水也就10万。这个市场太小，长期看发展空间有限。你能不能把'起步市场'定义为某个有放大潜力的细分入口？比如'武大文理学部大二的教材'既足够小可以占住，又有足够的相似度可以复制到其他学部和其他年级。"

### Q0.6 你的终极市场会比现在大10倍吗？

**Story**: 用户说："能。如果我在武大跑通了模式，可以复制到武汉其他高校。华科、武理工、华师，武汉有80多所高校。然后从武汉扩展到其他大学集中的城市，南京、西安、成都。目标覆盖全国双一流高校和主要本科院校，大概500到1000所，比起步的1所至少是两个数量级的增长。"

**Demo**: 代理追问："这个增长路径是线性的，一所学校一所学校啃，还是有杠杆的，比如校园大使、口碑传播、学生会合作？前者可能需要漫长的周期，你团队只有2到3人，融资能撑到那一天吗？"

**Acceptance**: 用户不仅说"能"，而且给出了从当前到10倍的具体增长路径。是扩地域、扩品类、还是扩客户群。路径有可信度，不是"世界那么大肯定能"。

**Error**: "不能"或者"没想过"。代理提醒："投资机构会非常关心你的增长空间。如果一个生意做大了也就现在的两倍规模，它不是一个可投资的故事。但这不意味着这个生意不值得做。如果你不需要融资，小而美的生意完全可行。你打算靠自有资金慢慢滚吗？"

**Edge**: 用户说"能增长100倍"但说不清楚怎么增长。代理不否定，但追问："100倍的规模需要完全不同的组织能力、资金结构和竞争策略。你幻想过那个规模下的公司长什么样吗？如果没想过，先聚焦10倍以内的增长路径会更现实。"

### Q0.6a 你是在创造一个全新的品类（像特斯拉开创了电动豪华车市场），还是在一个成熟的品类里和现有玩家抢份额？

**Story**: 用户说："我觉得是在创造一个微品类。单独说'二手交易'这个品类很成熟，但专门为校园教材服务的C2C匹配的、同校交易的平台，这个组合目前没有人在做。这是在一个大品类里切了一个全新的子品类。"

**Demo**: 代理判断这是"创造新品类"的方向（虽然是大品类中的细分创新），追问："创造新品类意味着你需要教育市场。用户习惯在闲鱼上买卖或者直接买新书，不知道有一个专门做校园教材交易的平台。你的资源够做市场教育吗？你准备用什么方法让学生意识到'有更靠谱的选择'？"

**Acceptance**: 用户诚实地判断了自己的品类定位。对于"创造新品类"的情况，用户对"教育市场"的成本有现实认知。对于"成熟品类竞争"的情况，用户能说出领头羊的弱点及自己的打击策略。

**Error**: "我是创造新品类"但实际描述出来就是市场上的一个现有产品。代理指出："你描述的功能xx公司已经在做了，这不叫新品类。新品类意味着市场分类表中还没有这一行。如果只是多了一个功能，你还是在一个成熟品类里竞争。请重新评估。"

**Edge**: 用户说"成熟品类竞争"但同时也指出了一些新模式。代理可以追问分支："你觉得现在的领头羊最怕什么？你有办法让它睡不着吗？"如果用户说"他们不愁客户因为他们太大"，代理追问："大公司的通病是什么？如果你的答案是'慢'，那你有没有利用速度优势打出时间差的具体计划？"

## 0D / 幂次法则与聚焦

### Q0.7 如果有且只能做一件事来让这个创业成功，那件事是什么？

**Story**: 用户想了很久，说："如果只能做一件事，那就是让平台上每个学校都覆盖足够多的教材品类和数量，做到每个学生来搜课号都能找到对应的二手书。因为一切，交易量、用户留存、口碑传播，都建立在'来了就能买到书'这个能力上。这件事做成了用户就不会走，其他事都可以找人做。"

**Demo**: 代理确认用户找到了真正的"命门"，然后推动执行层面的思考："好。这个标准非常具体。你判断自己覆盖到'来了就能买到书'需要多久？在这之前你打算怎么让用户容忍教材不全的阶段？另外你说其他都可以放一放。真的放得下吗？比如你完全没有市场推广能力，用户从哪知道你呢？"

**Acceptance**: 用户能说出唯一一件最核心的事，而不是把"做好产品"或"搞定客户"这种笼统的话当作答案。这件事确实是一切其他事情的基础，而不是"很重要但排第二"的事。

**Error**: 用户说了2到3件事（"做好产品、搞好销售、融到钱"）。代理追问："这些都是必须的，但幂次法则说的是，最重要的事比其余所有事加起来都重要。这三个里哪一个如果没做成，其他全部白费？那一件就是你当前唯一应该专注的事。"

**Edge**: 用户说不出来。代理不给答案，但用例子引导："想想你的生意中最脆弱的一环是什么。如果那个环节断了整个生意就转不了。可能是技术没搞定、可能是客户拿不到、可能是成本降不下来。那件事就是你的'唯一'。"

### Q0.8 你对未来的态度是什么？是认为未来可以被预测和规划，还是走一步看一步？

**Story**: 用户说："我属于明确乐观型。我相信校园交易的垂直化是一个清晰的趋势，大学生越来越习惯在专门的平台上解决特定需求。我甚至已经画了一张3年内每个学期要达成什么目标的路线图。"

**Demo**: 用户是"明确乐观"，适合精细规划。代理追问："你愿意把路线图的大致框架分享一下吗？不需要具体数字，但关键里程碑和时间节点我想看看。这会帮助我在后面的阶段里帮你校准节奏。"

**Acceptance**: 用户清晰地表达了自己对未来的基本态度，没有模棱两可。如果是"明确乐观"，用户能说清楚自己对未来的预测依据；如果是"明确悲观/不明确"，用户对自己需要什么帮助有清醒认知。

**Error**: "走一步看一步吧"。既不确定也不乐观。代理不批评，但指出影响："不明确的未来观会让你在规划时缺少方向感。我们后面的很多问题，市场选择、资源投入、融资时机，都需要你有一个大致的时间判断。我建议你至少在这个项目上做一个'有条件的预测'：在什么假设下你乐观，什么假设下你会悲观。"

**Edge**: 用户很哲学地说"未来是不可知的，KPI都是幻觉"。代理不辩论哲学，而是务实回拉："我理解不确定性是真实存在的。但创业的本质就是在不确定中做决策。你不需要预测10年，但你能不能说：如果我这个产品9月上线，到年底前我想看到什么结果？一个具体的、可以衡量的结果。"

## 0E / 特斯拉7问全景扫描

> 以下7问来自《从0到1》第13章"特斯拉的7个问题"。它们是战略完整性的快速体检，覆盖了技术、时机、垄断、团队、销售、持久、秘密七个维度。其中Q0.11（垄断）已在 0C 覆盖、Q0.15（秘密）已在 0B 覆盖，但此处作为全景扫描再次问一遍，目的是获得统一的状态灯评估。
>
> 处理方式：用户对每个问题做一句话回答。状态灯标记规则：
> - 绿灯：能给出具体、有依据的回答
> - 黄灯：回答模糊，需要更多思考
> - 红灯：无法回答或回答显示明显短板
>
> 红灯项记录并在对应Stage重点追问。

---

### Q0.9 技术：你的产品在核心技术上有突破性进步吗？进步幅度是10倍级的还是10%级的？

**Story**: 用户说："我们做了针对校园教材交易的C2C匹配系统，针对同一门课程不同教材版本做了专门优化。在实测中比闲鱼等通用平台的找书效率高很多。不是10倍级的突破，但在这个细分场景里算很大了，因为通用平台上你搜一本书可能要翻半天才能确认版本对不对。"

**Demo**: 代理判断是"10%级"但在这个垂直场景里是"显著差异"。标记绿灯，追问："匹配效率的提升是你自己测的还是对比了闲鱼的真实使用数据？如果是你自己跑的测试，样本量够不够？我需要帮你判断这个优势在学生面前有没有说服力。"

**Acceptance**: 用户量化了技术进步的幅度。不是空说"技术很牛"，而是给出具体的对比基准和提升比例。用户诚实面对"是10倍还是10%"的区分，没有夸大。更重要的是，用户能说出这个技术进步对客户意味着什么（不只是参数提升，而是解决了一个实际问题）。

**Error**: "技术上没什么特别的，就是整合现有技术"。但用户前面说这是"0到1"的创新。代理指出矛盾："你之前说你在做一个没人做过的东西，现在又说技术没突破。如果一个产品没有技术上或模式上的突破性进步，用户为什么要从现有方案切换过来？这个不匹配需要解释一下。"

**Edge**: 用户说"10倍级提升"但说不出具体对比基准。代理追问："10倍什么？速度快10倍？成本低10倍？准确率高10倍？和谁比？消费者真的在乎这个10倍吗，还是只是你技术团队觉得酷？"

---

### Q0.10 时机：为什么现在是做这件事的最佳时机？以前为什么没人做？

**Story**: 用户说："三个原因。第一，大学教材越来越贵。2025年很多新版教材定价已经过百，学生买二手的需求空前强烈。第二，现在的大学生完全习惯手机交易。微信支付、小程序下单没有任何学习成本。第三，之前类似尝试失败是因为没有解决信任问题（怕被骗、怕教材版本不对），而现在的小程序支付担保和评价体系已经成熟。现在是三个条件同时满足的时刻。"

**Demo**: 代理标记绿灯，追问反证："这三个条件哪个最脆弱？如果其中一个不成立，比如教育部出台政策强制教材降价，你还能撑过去吗？"

**Acceptance**: 用户给出了多层次的时机理由。不仅是技术成熟，还包括市场接受度、用户行为变化、基础设施条件。用户能说出"为什么是现在而不是3年前"。

**Error**: "因为我觉得是时候了"。没有具体的外部客观理由。代理追问："你的感觉背后有什么客观变化支撑？是政策变了、技术成熟了、用户习惯变了、还是竞争对手出问题了？创业时机需要外部条件的支撑，不只是内心的召唤。"

**Edge**: 用户说"以前也有人做过但失败了"但说不清为什么他们失败。代理追问："他们失败的原因如果仍然存在，你怎么解决？如果只是因为他们做得不够好而你做得更好，那确实也是一个时机理由：行业已经验证了需求，只是缺一个好的执行者。"

---

### Q0.11 垄断：你起步时能在一个小市场占据大份额吗？（参考Q0.4-Q0.6）

**Story**: 用户说："回到武大那个例子。5万在校生，我的目标是第一年覆盖20%的学生，也就是1万人用过我们的平台交易教材。在这个校园里，我就可以说是'占住了'。然后基于这个据点再复制到其他学校。"

**Demo**: 代理参照Q0.4-Q0.6的已有回答做一致性检查。如果一致，标记绿灯并说："你在Q0.5说过武大起步，在Q0.4选了网络效应作为护城河。现在你的数据和逻辑是自洽的。小市场、可防御的壁垒、明确的扩张路径。这部分我们已覆盖，没有红灯。"

**Acceptance**: 用户回答与之前的回答一致，没有出现前后矛盾。用户能说出具体的"小市场"定义和"大份额"的数字。

**Error**: 用户前后矛盾。前面说"起步市场是全国"现在又说"从小市场开始"。代理指出不一致："你在Q0.5说全国大学生教材市场是上百亿，现在又说可以从小市场开始。这两件事只能选一个。到底你的起步市场是一个具体的、可防御的小入口，还是一个广阔但竞争激烈的红海？"

**Edge**: 用户说Q0.4到0.6的问题时思路清晰，但这里突然开始自我怀疑。代理不压制这种怀疑，而是说："你已经有了很好的思考。自我怀疑是正常的，这些问题的目的不是让你100%确信，而是暴露哪些地方还需要更多信息。"

---

### Q0.12 团队：你和你团队有做这件事的独特优势吗？

**Story**: 用户说："我和联创都是武大毕业的，一个做过校园市场的运营，一个带过校园团队。我们太了解学生的行为习惯了。他们在哪里找信息、怎么付钱、什么情况下愿意信任陌生人。这可能是我们的独特优势。不是技术最强，但最懂这个场景。"

**Demo**: 代理先肯定团队的匹配度，然后追问潜在盲区："你们的背景高度重合。都是武大校友、都有校园运营经验。有没有缺少的关键角色？比如技术合伙人还在外面，你们打算外包还是找CTO？如果平台开发是核心壁垒，技术短板怎么补？"

**Acceptance**: 团队背景与创业方向直接相关，用户能诚实说出自己的强项和弱项，而不是"我们团队完美无缺"。对于关键短板有明确的弥补计划。

**Error**: "我们三个大学生，啥都不懂但学习能力很强"。代理标记红灯："学习能力强不等于有行业认知。你的生意需要懂校园市场、懂学生心理、懂平台运营。这些领域任何一个都可以让你交很贵的学费。你打算怎么加速学习？有没有行业顾问？"

**Edge**: 用户是单人创业者。代理追问："彼得·蒂尔说创始团队像帮派，需要多元但一致的几个人。你一个人做所有事情在初期可能可以，但你有没有想过哪些角色必须有人分担？如果融到资你第一个招谁？"

---

### Q0.13 销售：你怎么让用户知道并买到你的产品？

**Story**: 用户说："计划是先做校园大使。在每个学院找1到2个学生做推广，开学季在教学楼和食堂门口设点宣传，同时在班级群和校园集市发帖。这行还是靠同学之间的信任和口碑传播，线上广告在学生群体里效果不如熟人推荐。"

**Demo**: 代理追问获客成本和转化率假设："你的客单价大概多少？如果一单平均赚5到10元佣金，需要多少交易量才能覆盖运营成本？一个校园大使一个月能拉来多少用户？按行业标准校园大使转化率算，这个速度和你的增长预期匹配吗？"

**Acceptance**: 用户给出了具体的获客路径。不是"靠口碑"或"靠广告"。用户对"怎么把产品送到用户手上"有实际的操作思路，而不是停留在"产品好自然会有人买"的阶段。

**Error**: "靠口碑传播"。代理说："口碑是结果不是策略。一个冷启动的产品没有口碑可言。你怎么触发第一波用户？他们怎么知道你？是你去告诉他们，还是他们自己搜到你？如果是后者，你打算做什么让他们能搜到？"

**Edge**: 用户说"和行业协会合作"但还没谈过。代理标记黄灯，追问："和协会合作是个好方向，但你要想清楚。协会为什么要帮你推广？你给协会什么好处？这个事情处于'想'阶段还是已经在谈了？"

---

### Q0.14 持久：10年后你的市场地位还稳吗？什么力量可能颠覆你？

**Story**: 用户说："10年后我们的护城河应该是学校网络效应和数据壁垒。到时候我们覆盖了几百所高校的教材交易数据。什么课用什么教材、哪些版本有二手市场、每个学校的教材流转周期。这些数据会让新进入者几乎不可能追平。但最大的威胁可能是巨头。闲鱼如果决定做校园细分频道，他们有现成的用户和资金。"

**Demo**: 代理肯定了用户看到了长期壁垒，也识别了威胁，追问应对策略："你说了巨头威胁。如果闲鱼在2年后进入这个市场，你的网络效应领先优势是2年。2年后你可能也就覆盖了几十所学校。你觉得2年的先发优势够做护城河吗？还是你需要在2年内建立一种运营壁垒？比如每所学校的课程清单和教材映射关系需要一家一家维护，迁移成本很高。"

**Acceptance**: 用户不止说了"我的生意很稳"，而且诚实地审视了可能颠覆自己的力量。不管是大公司、新技术、还是市场变化。用户对威胁的应对不是一个浅薄的"我们做得更好"，而是有具体的防御策略。

**Error**: "没想过"或者"不会有人能颠覆我们吧"。代理标记红灯："彼得·蒂尔问'10年后你的市场地位稳吗'是在逼你思考防守。如果任何有资源的大公司都可以进入你的市场然后打败你，你的生意没有长期价值。你不需要现在就完美回答，但你需要开始思考这个问题。"

**Edge**: 用户说"10年后这行业可能不存在了"。这个回答在Z2O框架里看起来是消极的，但可能代表深刻的洞察。代理不批评，追问："如果你认为行业可能不存在，那你的计划是做到一定规模就卖掉？还是说你认为你的产品是这个转型过程中的关键环节？这里可能会影响你后面融资时的故事方向。"

---

### Q0.15 秘密：你发现了什么别人没看到的独特机会？（参考Q0.3）

**Story**: 用户说："大公司都在做综合二手交易平台，但校园教材这个细分市场大家觉得太小、运营太重、利润太低，不愿意做。我的观察是，它被低估了。校园市场虽然分散在不同学校，但每个学校的需求高度标准化。每个学期、每门课用什么教材，清单是确定的。用一个标准化的平台去服务一个被低估的细分市场，这就是我的秘密。"

**Demo**: 代理参照Q0.3的回答做一致性检查。如果一致，标记绿灯并说："你在Q0.3说了同样的逻辑，需求空白型的秘密。前后一致，没有红灯。不过你在Q0.14提到了闲鱼威胁，如果闲鱼某天改变主意进入这个细分市场，你现在的秘密就不存在了。你还有没有更深的、不容易被复制的秘密？"

**Acceptance**: 用户回答与Q0.3一致，没有前后矛盾。秘密声明不是一个编出来的故事，而是用户真正相信的东西。

**Error**: 用户在这里和Q0.3说了不一样的东西。代理指出："你在Q0.3说你的秘密是'需求空白，大众平台不做校园教材细分'，现在又说秘密是'校园教材需求被低估'。这两个都可以是秘密，但它们是不同的秘密。最核心的那个秘密到底是什么？"

**Edge**: 用户说"我已经说了3遍了"。代理不照本宣科，回应："确实，这部分我们在Q0.3已经深入讨论过。你的答案是XX，和当时一致。我只是再确认一遍，经过前面10个问题的讨论后你没有改变想法。保持一致是好事，说明你的核心逻辑是稳固的。"

## 检查点：是否值得深入？

### QCheck-0 基于以上7问，你觉得这个想法值得花时间深入测试吗？还是需要先做更多调研/放弃？

> 这不是一个"答题"环节，而是一个决策时刻。代理汇总特斯拉7问的状态灯给用户看，然后让用户自己决定。

**Story**: 用户看到状态灯：技术绿、时机绿、垄断黄（护城河还不稳）、团队绿、销售黄（需要验证转化率）、持久黄（巨头威胁）、秘密绿。用户说："看起来大部分是亮的，但护城河和持久这块我还有点不确定。不过我觉得值得继续测，至少走到Stage 1先做个系统的机会评估。"

**Demo**: 代理不替用户做决定，但提供决策支持："你的7问里有3个黄灯，集中在防御端（护城河、持久）和销售端。这些恰恰是Stage 1（机会评估）和Stage 4（增长/销售）要深入的方向。如果你现在停下来，你会浪费这些初步思考的势能。我建议先标记这些黄灯，在后面的Stage里逐个解决。准备好了我就推进到Stage 1。"

**Acceptance**: 用户基于手上的信息做了一个清醒的决定。无论是"继续"、"先调研"还是"放弃"。决定不是冲动，而是权衡了风险和机会之后的选择。

**Error**: 用户明明有多个红灯但说"我觉得没问题继续吧"。代理指出："你有X个红灯，XX、XX，这些是硬伤。如果你不做额外调研就继续，这些短板会在后面反复出现。我建议你至少先解决这些红灯再做深入评估，不然Stage 1到5的很多问题你答不上来。"

**Edge**: 用户说"今天没状态，我改天再决定"。代理不强迫，记录下来当前进度，说："没问题，这是一个重要的决定，值得认真考虑。你目前已经完成了Stage 0的全部问题，所有回答都已记录。当你准备好了，叫我继续。"
