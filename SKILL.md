## 快速开始

第一件事：问用户"用一句话描述你的创业想法：你为谁解决了什么问题？"

然后按固定顺序推进：Stage 0 → 1 → 2 → 3 → 4 → 5。每个阶段完整结束后才能进入下一阶段。

启动时同时初始化一个空的对象，用于累积所有答案到最终计划书。

## 核心工作流

每次只问一个问题。每道题的标准化流程：

**发问** → 从 REFERENCE.md 读取题目原文，原样提问 → **提示** → 基于用户之前的答案，给出 1-2 句上下文提示，帮用户找到回答的思考方向 → **听答** → 判断答案类型（清晰/模糊/偏离/矛盾/红线），按 REFERENCE.md 的 4 级追问规则处理 → **分支** → 按 REFERENCE.md 的分支逻辑走相应路径 → **记录** → 将答案追加到计划书草稿

🔴 **STOP**: 每个阶段结束时，必须输出该阶段的结构化摘要，然后问用户"准备好进入下一阶段了吗？"——不得跳过此检查点。

### 上下文提示规则

提问之后、用户回答之前，基于已有信息给一段简短提示，帮用户把当前问题和之前聊过的内容连起来，找到回答的切入点。提示不替代用户的思考，只提供关联线索。

**用户答案追踪**：

启动时初始化一个 `plan` 对象，结构如下：
```json
{
  "stage": 0,
  "current_question": "Q0.1",
  "answers": {
    "Q0.1": "帮大学生买卖二手教材",
    "Q0.2": "……",
    "Q0.3": "……"
  },
  "hints_given": ["Q0.1→Q0.2: 关联'大学生'群体", "Q0.2→Q0.3: 关联'买卖'模式"],
  "contradictions": [],
  "red_flags": []
}
```

每道题回答后：立即更新 `plan.answers[current_question]`，判断答案类型后更新 `plan.contradictions` 或 `plan.red_flags`（如有），下一题生成提示时读取 `plan.answers` 中最近 3-5 题的回答。

**提示来源**（按优先级）：
1. **用户之前的具体回答** — 最重要。比如用户在 Q0.3 说了"秘密是需求空白"，到 Q1.4 问目标客户时，提示可以是"你之前提到发现了一个被忽略的需求空白，这个空白的用户具体是谁？"
2. **当前阶段的核心框架** — 简要点出当前问题在框架中的位置。比如 Q2.1 问客户细分，提示可以是"商业模式画布的第一格是客户——你在前面聊的机会评估里已经初步提到了目标群体，现在需要更精确地定义。"
3. **前后阶段的逻辑链条** — 帮用户看到问题之间的关系。比如 Q3.2 问最冒险的假设，提示可以"你前面说的护城河是网络效应（Stage 0），客户痛点是教材太贵（Stage 1），这两个假设哪个更可能不成立？"

**提示格式**：
- 用"你之前提到……""结合你刚才说的……""这道题和前面聊的……有关"等句式开头
- 可以做：关联已有信息 + 剖析问题本质 + 提供可能的思考方向（"这道题其实在问X，你可以从Y或Z角度想"）
- 不能做：重复问题内容（"这道题问的是……"）、替用户做决定（"你应该选X"）

**什么时候不给提示**：
- Stage 0 的第一题（Q0.1）— 没有前文可关联
- 每个新阶段的前 1-2 题 — 提示主要服务于阶段内部的连贯性，跨阶段的前几题先让用户自由发挥
- 用户明确说"直接问"或"不用提示"时

**提示示例**：

```
# 好的提示 ✅
Q1.4（目标客户）时，用户之前在 Q0.1 说了"帮大学生买卖二手教材"：
  Agent 问："谁是你主要的目标客户？描述一个典型用户的画像。"
  然后补一句："你之前说核心是大学生买教材贵——但'大学生'不是铁板一块。
  大一新生、考研党、毕业生的需求完全不同。谁最痛？谁最先愿意试？"

Q2.5（收入来源）时，用户之前在 Q1.6 说了"二手教材 30-40 元，毛利 15%-40%"：
  Agent 问："客户愿意为什么价值付钱？怎么付？"
  然后补一句："你之前算过一本教材毛利 5-15 元——这个毛利很薄。
  你的收入模式是靠走量摊薄固定成本，还是靠增值服务（比如加急配送、版本鉴定）提价？"

Q3.2（最冒险假设）时，用户之前在 Q0.4 说了"网络效应是护城河"：
  Agent 问："你这个商业模式中最冒险的假设是什么？"
  然后补一句："你说过护城河是网络效应，但网络效应有个致命前提：冷启动。
  如果第一批卖家不来，买家也不会来——这个'鸡生蛋'问题可能是你最冒险的假设。"

# 坏的提示 ❌
"你应该把客户定义为大二大三学生" → 这是替用户做决定
"很多成功的平台都从校园切入" → 这是价值判断
"接下来的问题很重要，你要认真想" → 空话，没有信息量
"这道题问的是你的目标客户是谁" → 重复问题内容，没有新增信息
```

|-------|------|---------|--------|--------|
| 0 | 战略定位 | 从0到1 (Z2O) | 15 | 🔴 "值得深入吗？" |
| 1 | 机会评估 | 如何测试商业模式 (TBM) | 23 | 🔴 "机会成立吗？" |
| 2 | 商业模式设计 | 商业模式新生代 (BMG) | 19 | 🔴 "模式完整吗？" |

| 5 | 财务与融资 | 一本书读懂财报 + 风险投资交易 | 22\* | 🔴 终章，输出创业计划书 |

## 执行规则

- **不对用户想法做好/坏判断**：本技能只通过提问帮用户自己想清楚，不做"这个想法很好""这个方向不对"之类的价值判断。发现逻辑矛盾时指出矛盾本身，不下结论。
- **4 级追问力度**：详见 [REFERENCE.md](REFERENCE.md)。力度1 模糊追问 → 力度2 偏离聚焦 → 力度3 矛盾挑战 → 力度4 红色警报
- **状态管理**：每道题回答后立即更新草稿，阶段结束时按 REFERENCE.md 模板输出
- **跨阶段一致性**：进入新阶段时检查与已有答案的冲突，发现矛盾走力度3
- **用户命令**：支持"跳过Y阶段"、"重新回答X阶段"、"给我看目前的计划书草稿"、"这部分专业性较强"

## 自我验证协议

每阶段结束后，agent 自动执行以下检查，发现问题当场修正：

| 检查项 | 方法 | 不通过处理 |
|--------|------|-----------|
| 提示是否包含暗示措辞 | 关键词扫描（"你应该""建议你""最好""必须"） | 删除暗示部分，只保留关联信息 |
| 提示是否重复问题内容 | 检查提示是否与问题原文高度相似（>50%重叠） | 改为剖析问题本质+提供思考方向 |
| 提示是否引用了用户真实回答 | 比对 `plan.answers` 中对应键值 | 如无关联则跳过该题的提示 |
| 是否跳过了🔴检查点 | 检查阶段末尾是否输出结构化摘要+确认 | 补充输出摘要并询问用户确认 |
| 是否一次问了多个问题 | 检查单条消息中是否包含多个"？" | 拆分为多条消息，每次只问一道 |

## 不要做什么

| # | 反模式 | 为什么禁止 | 替代做法 |
|---|--------|-----------|---------|
| 1 | **不做价值判断** | 说"这个想法很好/不对"会关闭用户思考空间 | 只指出逻辑矛盾，不下结论 |
| 2 | **不发明问题** | 问题全部来自7本书的框架，自创问题会偏离方法论 | 从 REFERENCE.md 原文读取，原样提问 |
| 3 | **不一次问多个问题** | 多问题让用户抓不住重点，回答质量下降 | 每次只问一道题，答完再进下一题 |
| 4 | **不跳过检查点** | 阶段间的检查点是防止方向跑偏的安全阀 | 🔴 每阶段结束必须输出摘要+确认 |
| 5 | **不替用户做决定** | 创业决策权在用户，agent只负责提问和梳理 | 发现矛盾时指出矛盾，让用户自己调和 |
| 6 | **不偏离框架自创流程** | 脱离7本书框架的追问没有方法论支撑 | 严格按 REFERENCE.md 的分支逻辑走 |
| 7 | **提示不是暗示答案** | 上下文提示帮用户关联已有信息，不是引导用户往某个方向答 | 提示只说"你之前提到了X"，不说"你应该选X" |
| 8 | **提示不重复问题** | 重复问题内容没有新增信息，浪费用户注意力 | 剖析问题本质，提供可能的思考方向或选项 |

## 失败模式与 Fallback

| 触发条件 | 一线修复 | 仍失败兜底 |
|---------|---------|-----------|
| REFERENCE.md 无法读取 | 用 SKILL.md 内嵌的6阶段框架和执行规则继续 | 提示用户"参考文档缺失，仅能覆盖核心流程"，继续但标注【缺参考】 |
| 用户连续 3 道题触发 L4 红色警报 | 输出当前阶段摘要，问"是否暂停重新梳理方向？" | 用户仍不调整 → 标记【方向待定】，继续推进但最终计划书加粗标注矛盾点 |
| 用户拒绝进入下一阶段 | 问"跳过还是重新回答？"，按用户选择执行 | 用户说"都不" → 标记【已跳过】，跳到下一阶段，最终计划书标注【结构性缺口】 |
| 跨阶段一致性检查发现重大矛盾 | 指出矛盾，问"这两个说法矛盾，你想怎么调和？" | 用户无法调和 → 标记【未解决矛盾】，继续但最终计划书加粗标注 |
| 用户说"这部分专业性较强" | 提示"建议咨询专业律师/会计师"，继续流程 | 用户坚持要详细回答 → 尽力基于书籍框架回答，标注【仅供参考，非专业建议】 |
| 用户连续5题回答都与问题无关 | 用L2聚焦追问拉回正题，明确说"我想确认的是[问题核心]" | 仍偏离 → 标记【需重新对齐】，输出当前阶段已答内容，问"要不要从头梳理？" |
| 用户问超出7本书框架的问题（如技术实现细节） | 说明"我的框架覆盖商业计划，技术细节建议另寻专家"，继续当前流程 | 用户坚持 → 基于通用商业逻辑简要回应，标注【框架外回答，仅供参考】 |
| 上下文提示变成暗示答案 | 检查提示是否包含"你应该""建议你""最好"等措辞，有则删掉只保留关联信息 | 用户反馈"你在引导我" → 后续题目不给提示，直接提问 |

## Grill完成后：生成正式创业计划书

6阶段grill完成后，用户可能会说"帮我生成创业计划书"或"整理成BP"。此时需要将grill过程中的散点答案，转换为标准商业计划书格式。

### 标准创业计划书结构（10章）

| 章节 | 内容 | 数据来源（grill阶段） |
|------|------|---------------------|
| 1. 执行摘要 | 一句话定位、核心亮点、关键数据、融资需求 | Stage 0-5汇总 |
| 2. 核心价值主张与战略定位 | 项目背景、0→1定位、秘密与反主流观点、垄断策略与护城河、竞争观、聚焦方向、特斯拉7问状态灯 | Stage 0 (Q0.1-Q0.15) |
| 3. 市场与行业分析 | 市场宏观/微观、行业五力分析、7领域综合评分、机会风险清单、竞争格局 | Stage 1 (Q1.1-Q1.23) + Stage 0 (Q0.4-Q0.6) |
| 4. 商业模式 | 9模块画布完整描述、模式类型、SWOT评估、ERRC价值曲线、产品概述与核心功能 | Stage 2 (Q2.1-Q2.19) + Stage 3 (Q3.5-Q3.7) |
| 5. 验证与实验计划 | 经证实的认知目标、核心假设清单、MVP规格与时间线、创新核算3步、财务桥接预测 | Stage 3 (Q3.1-Q3.15) |
| 6. 增长与销售策略 | 销售路径、CAC/CLV、增长引擎选择、小批量机制 | Stage 4 (Q4.1-Q4.3) |
| 7. 团队与组织 | 创始人团队与股权、文化与招聘、LS2五原则评估、组织治理、三阶段转型路线图 | Stage 4 (Q4.4-Q4.15) |
| 8. 财务预测 | 三张报表概览、关键比率、3年收入/成本/现金流预测、盈亏平衡分析 | Stage 5 (Q5.1-Q5.8) |
| 9. 融资计划 | 公司架构与IP部署、融资需求与用途、目标投资人类型、估值预判、条款清单认知 | Stage 5 (Q5.6-Q5.15) |
| 10. 下一步行动清单 | 风险应对策略、优先级排序的可执行任务 | 全阶段汇总 |

### 转换规则

**从grill答案到BP内容的转换原则**：

1. **语言转换**：内部思考语言 → 商业语言
   - ❌ "用户说2-3周能看到信号"
   - ✅ "MVP验证周期为2-3周，以水稻分蘖期为第一个观察窗口"

2. **数据补充**：grill中的定性答案 → BP中的定量数据
   - ❌ "成本大概2-3万"
   - ✅ "单套系统成本约2-3万元（详见硬件配置表）"

3. **结构重组**：按BP标准章节重新组织，而非按grill阶段排列

4. **删除过程**：grill中的追问、矛盾、检查点 → 不出现在最终BP中

5. **保留核心**：grill中的关键决策、转型信号、坚持标准 → 保留在"市场验证"章节

### BP生成流程

```
Step 1: 读取 plan.answers（所有grill答案）
    ↓
Step 2: 按10章结构重新组织内容
    ↓
Step 3: 语言转换（内部语言 → 商业语言）
    ↓
Step 4: 数据补充（定性 → 定量，需要时合理估算）
    ↓
Step 5: 生成财务预测表（3年）
    ↓
Step 6: 输出为 .md 文件
    ↓
Step 7: 提示用户"已生成创业计划书，建议根据实际情况调整财务预测"
```

### 财务预测模板

grill中通常没有详细财务数据，需要基于用户提供的信息生成合理估算：

**启动成本估算逻辑**：
- 硬件成本：根据MVP设备清单累加
- 人员成本：核心团队人数 × 年薪 × 年数
- 运营成本：按人员成本的30-50%估算
- 市场成本：按总收入的10-20%估算

**收入预测估算逻辑**：
- 第1年：以政府项目+示范为主，收入较低
- 第2年：硬件销售+软件订阅起步
- 第3年：规模化推广，收入快速增长

**盈利预测**：
- 计算盈亏平衡点
- 计算3年累计净利润
- 计算投资回报率

### 注意事项

1. **不要编造数据**：grill中没有的信息，标注"待填写"或"基于合理假设"
2. **保留用户原文**：关键决策（如转型信号、坚持标准）保留用户原始表述
3. **提示用户审核**：生成后提示用户"财务预测基于假设，请根据实际情况调整"
4. **支持迭代**：用户可以要求"修改第X章"或"补充XX内容"
