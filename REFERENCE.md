# startup-plan-grill 参考文档

## 执行协议

### 答案分类与处理

| 答案类型 | 判断标准 | 处理 |
|---------|---------|------|
| 清晰具体 | 直接回答了问题 | 记答案，进分支逻辑或下一题 |
| 模糊笼统 | "还行吧"、"到时候再说" | 力度1 温和探询 |
| 答非所问 | 未触及问题核心 | 力度2 聚焦追问 |
| 与之前矛盾 | 与同一/之前阶段冲突 | 力度3 挑战检验 |
| 触及红线 | 3次+矛盾 / 红线条件触发 | 力度4 红色警报 |

分支有明确路径按路径走；含"红灯"警告必须提示；指向特定追问则追问一次后继续。

### 4 级追问力度

**L1 温和探询** — 答案模糊。追问一次（"能具体说说吗？"）。仍模糊→标记已追问，进下一题。
**L2 聚焦追问** — 答非所问。追问一次拉回正题。仍偏离→标记后进下一题。
**L3 挑战检验** — 逻辑矛盾。指出矛盾→听解释→能调和则记录，不能则标记"未解决矛盾"。
**L4 红色警报** — 3次+矛盾 / 题目标记严重警告（如Q0.4） / 盲目乐观。提醒用户认真考虑，可选暂停或继续。

### 状态与输出管理

Agent 维护累积 object，每道题后更新：
```
计划书草稿: { Stage0: { answers: { Q0.1: "..." }, summary: "" }, ... }
```

特斯拉7问红绿灯：Stage 0 记录各维度状态（绿=有依据 / 黄=模糊 / 红=短板），对应 Stage 完成时重新评估更新。状态变化在阶段总结中提示。

阶段完成→按该阶段输出模板生成 summary→问"准备好进入下一阶段了吗？"

**命令**：
- "查看草稿" → 按10章模板组装已答内容，未答=【待定】，跳过=【已跳过】，不改变进度
- "重新回答X阶段" → 清空该阶段 answers 和 summary，回到第一题
- "跳过Y阶段" → 标记【已跳过】，继续下一阶段。跳过关键阶段（0/3）时提示可能存在结构性缺口

### 跨阶段一致性检查

进入新阶段时扫描常见冲突：平台极窄市场、不自融却融资、免费无成本支撑、数据与预测脱节、病毒式配窄人群、兼职配高增长。发现矛盾走力度3，不强行阻止。

## 跨书张力

设计文档明确了三组跨书张力及其处理方式。代理在运行时如遇到用户对框架方法论的疑惑，应使用以下解释：

### Z2O vs LS 哲学冲突

Z2O 主张"秘密→垄断"的确定论规划路线；LS 主张"假设→验证"的试错路线。

**处理方式**: Z2O 作为战略假设（Stage 0），LS 作为验证方法（Stage 3），二者不矛盾。先提出垄断假设，再用精益方法验证。

### TBM vs BMG 侧重不同

TBM 侧重外部机会评估，BMG 侧重内部模式设计。

**处理方式**: 先评估机会（Stage 1），确认值得做之后，再设计模式（Stage 2）。如果机会不成立，模式再漂亮也没用。

### LS vs LS2 重复

LS 的创新核算（3 步）和 LS2 的创新核算（3 层次）是同一概念的不同细化。

**处理方式**: LS 的 3 步核算放在 Stage 3（MVP 验证阶段）；LS2 的 3 层次放在 Stage 4（组织治理阶段）。

## Stage 0: 战略定位 / 这到底是个什么想法？

**目标**: 帮用户用 Z2O 的框架审视自己的想法，判断是否值得深入。

**时长预估**: 5-8 轮问答

**参考书籍**: 从0到1 Ch1 + Ch3 + Ch4 + Ch5 + Ch6 + Ch7 + Ch13

### 题目

#### 0A / 热身：一句话想法

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q0.1 | 用一句话描述你的创业想法：你为谁解决了什么问题？ | 说不清 → 追问"客户是谁"和"什么问题"；说清了 → 进 Q0.1a |
| Q0.1a | 你这个想法是"从0到1"（创造一种前所未有的东西）还是"从1到n"（复制已有的成功模式，但做得更好或更便宜）？ | 从1到n → 追问"你凭什么认为你能比现有玩家做得更好？如果只是更好一点，用户为什么要换？"；从0到1 → "那意味着你相信一个别人没看到的秘密，进 Q0.2" |

#### 0B / 秘密与反主流观点

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q0.2 | 在什么重要问题上，你与绝大多数人有不同看法？ | 回答与创业无关 → 追问"这对你的生意意味着什么"；说不出 → 红灯"没有独特观点的生意很难垄断" |
| Q0.3 | 你这个想法是基于一个什么样的"秘密"？是别人不相信但你知道的技术真相，还是别人没注意到的用户需求空白？ | 自然秘密 → 走技术路线；人的秘密 → 走需求路线；没有秘密 → 建议重新审视 |
| Q0.3a | 你怎么看待竞争？很多人说"有竞争说明市场好"，但也有人说"竞争是利润的毁灭者" | 正面看待竞争 → "小心：如果赛道已经挤满了人，你的'秘密'可能根本不是秘密"；负面看待竞争 → "你打算怎么避开竞争，创造垄断？进 Q0.4" |

#### 0C / 垄断与护城河

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q0.4 | 你的产品/服务靠什么保护自己不被模仿？选出最匹配的一项：专利技术 / 网络效应 / 规模经济 / 品牌优势，或者以上都不是？ | **以上都不是** → **严重警告"没有护城河的生意容易被巨头碾压"** |
| Q0.5 | 你的起步市场是多大？具体说，你第一个打入的市场是多小的？ | 市场太大 → 警告"大市场竞争惨烈，建议从细分小市场开始"；太小且无增长可能 → 警告"天花板太低" |
| Q0.6 | 你的终极市场会比现在大 10 倍吗？ | 不能 → 提醒"投资机构会关心增长空间" |
| Q0.6a | 你是在创造一个全新的品类（像特斯拉开创了电动豪华车市场），还是在一个成熟的品类里和现有玩家抢份额？ | 创造新品类 → "需要教育市场，资源够吗？"；成熟品类竞争 → "你觉得现在的领头羊最怕什么？你有办法让它睡不着吗？" |

#### 0D / 幂次法则与聚焦

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q0.7 | 如果有且只能做一件事来让这个创业成功，那件事是什么？ | 说不清 → "幂次法则：最重要的事胜过其余总和"；说多个 → 追问哪个最决定成败 |
| Q0.8 | 你对未来的态度是什么？是认为未来可以被预测和规划，还是走一步看一步？ | 明确乐观 → 适合精细规划；不明确 → 需要先做更多调研 |

#### 0E / 特斯拉7问全景扫描（Z2O Ch13）

在深入各阶段之前，用彼得蒂尔的"特斯拉7问"快速扫描你的想法的完整性。这7个问题覆盖了后面所有Stage要深挖的方向。用户对每个"待定"问题做一句话回答。说不清的标记为"红灯"，在后面对应的Stage重点追问。

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q0.9 | 技术：你的产品在核心技术上有突破性进步吗？进步幅度是10倍级的还是10%级的？ | 对应 Stage 1/2 (待定) |
| Q0.10 | 时机：为什么现在是做这件事的最佳时机？以前为什么没人做？ | 对应 Stage 1 (待定) |
| Q0.11 | 垄断：你起步时能在一个小市场占据大份额吗？（参考 Q0.4-Q0.6） | 已在 0C 覆盖 **(已覆盖)** |
| Q0.12 | 团队：你和你团队有做这件事的独特优势吗？ | 对应 Stage 4 (待定) |
| Q0.13 | 销售：你怎么让用户知道并买到你的产品？ | 对应 Stage 4 (待定) |
| Q0.14 | 持久：10年后你的市场地位还稳吗？什么力量可能颠覆你？ | 对应 Stage 5（财务中的风险评估）(待定) |
| Q0.15 | 秘密：你发现了什么别人没看到的独特机会？（参考 Q0.3） | 已在 0B 覆盖 **(已覆盖)** |

**说明**: Q0.11 和 Q0.15 标记为"已覆盖"，代理无需再追问答案，但应在输出模板中列出状态。

用户选择继续则进入 Stage 1，否则建议先补充调研。代理不应强迫用户进入下一阶段。

## Stage 1: 机会评估 / 这个想法经得起检验吗？

**目标**: 用 TBM 的 7 领域模型做全面体检，判断机会是否值得追求。

**时长预估**: 10-15 轮问答

**参考书籍**: 如何测试商业模式 Ch1-Ch8

### 题目

#### 1A / 市场宏观

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.1 | 你进入的市场总规模有多大？（人数/金额/数量）如果不知道，打算怎么估算？ | 直接提问 |
| Q1.2 | 这个市场过去 3 年增长多快？未来 3 年预测增速？ | 直接提问 |
| Q1.3 | 有哪些大趋势在推动或阻碍这个市场？（人口/技术/社会/政策/环境） | 直接提问 |

#### 1B / 市场微观 + 移情图

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.4 | 谁是你主要的目标客户？描述一个典型用户的画像（年龄、身份、场景、痛点） | 直接提问 |
| Q1.5 | 这个客户现在用什么替代方案解决他的问题？他对现状满意吗？ | 直接提问 |
| Q1.6 | 客户愿意为你的解决方案付多少钱？这个价格覆盖得了成本吗？ | 直接提问 |
| Q1.7 | （移情图追问）客户嘴上说想要什么，和他实际做的有什么不一致？不一致的地方往往就是机会 | 直接提问 |

#### 1C / 行业宏观（五力分析）

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.8 | 这个行业新进入者容易进来吗？进入壁垒有哪些？ | 直接提问 |
| Q1.9 | 供应商的议价能力强吗？你依赖少数的供应商还是有很多选择？ | 直接提问 |
| Q1.10 | 购买者（你的客户）议价能力如何？他们能轻松换别家吗？ | 直接提问 |
| Q1.11 | 替代品的威胁有多大？客户有别的办法解决同样的问题吗？ | 直接提问 |
| Q1.12 | 行业目前的竞争有多激烈？有几家主要玩家？ | 直接提问 |

#### 1D / 行业微观（竞争优势）

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.13 | 你的利基（niche）在哪？能在行业中找到一个可防御的细分位置吗？ | 直接提问 |
| Q1.14 | 你的竞争优势能持续多久？靠什么保持？（专利/品牌/规模/数据/网络效应） | 直接提问 |
| Q1.15 | 你算过基本单位经济学吗？单个客户卖多少，直接成本（COGS）是多少？毛利率够不够覆盖后面的销售和管理费用？ | 直接提问（注意：CAC/CLV 将在 Stage 4 中深入计算，此处不做要求） |

#### 1E / 使命与个人

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.16 | 你为什么做这件事？纯粹赚钱还是更深层的使命驱动？ | 直接提问 |
| Q1.17 | 如果 5 年不赚钱，你还会继续做吗？ | 直接提问 |
| Q1.18 | 你愿意为这个创业付出多大代价？（时间、金钱、生活方式） | 直接提问 |

#### 1F / 执行能力

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.19 | 这个行业成功的关键要素是什么？（技术/渠道/品牌/资金/人才） | 直接提问 |
| Q1.20 | 你和团队在哪些关键要素上有优势？哪些是短板？ | 直接提问 |
| Q1.21 | 短板能通过招人/学习/合作补齐吗？ | 直接提问 |

#### 1G / 关系网络

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q1.22 | 你和团队在这个行业里有人脉吗？能打开关键渠道吗？ | 直接提问 |
| Q1.23 | 有没有潜在的导师/顾问可以帮你少踩坑？ | 直接提问 |

## Stage 2: 商业模式设计 / 怎么做成这件事？

**目标**: 设计完整的商业模式画布，匹配模式类型，评估短板。

**时长预估**: 12-18 轮问答

**参考书籍**: 商业模式新生代 全书（画布9模块 + 5模式类型 + SWOT + 蓝海战略ERRC）

### 题目

#### 2A / 画布 9 模块（依次扫描）

按顺序逐格完成，每格确认后再进下一个。

| # | 问题 | 分支逻辑 |
|---|------|---------|
| Q2.1 | [CS] 我们在为谁创造价值？可分为几个细分群体？谁最重要？ | 直接提问 |
| Q2.2 | [VP] 我们帮客户解决什么问题？满足什么需求？这是"救命的药"还是"维生素"？ | 直接提问 |
| Q2.3 | [CH] 客户希望怎么接触到我们？从认知到评估到购买到传递到售后，哪个阶段最薄弱？ | 直接提问 |
| Q2.4 | [CR] 每个细分期待什么类型的关系？获取成本高还是留存成本高？ | 直接提问 |
| Q2.5 | [R$] 客户愿意为什么价值付钱？怎么付（买断/订阅/租赁/中介费/广告）？定价策略？ | 直接提问 |
| Q2.6 | [KR] 最需要什么才能让生意运转（实体/知识/人力/资金）？ | 直接提问 |
| Q2.7 | [KA] 每天必须做的核心事是什么（生产/问题解决/平台网络）？ | 直接提问 |
| Q2.8 | [KP] 关键合作伙伴是谁？他们提供什么资源？ | 直接提问 |
| Q2.9 | [C$] 最贵的成本是什么？成本驱动还是价值驱动？ | 直接提问 |

#### 2B / 模式类型匹配

| # | 问题 | 分支逻辑 |
|---|------|---------|

| Q2.11 | 你是做客户原来自己做的事（替他们省事），还是做他们做不到的事？ | 直接提问 |

| # | 问题 | 分支逻辑 |

#### 2D / 蓝海战略（ERRC 四项行动框架）

基于你对行业竞争格局的理解（Stage 1 输出），用蓝海战略的 ERRC 框架重新审视你的价值主张：
